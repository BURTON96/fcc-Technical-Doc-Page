Introduction to Git

Git is an incredibly popular version control stystem and is one of the most used. Git is an actively maintaine open source project developed in 2005 by Linus Torvald, the creator of Linux operating system kernel.

GitHub, which is commonly confused for Git, is a service that allows us to upload code projects to Git and manage our code in a easy to use web interface.

Below you will find outlined details on how to configure both Git and GitHub to your personal device. I will also outline some quick commands you can perform from the command line to aid you in any projects you may be working on.

Step 1: Install Git for MacOS

Step 1.0: Install Homebrew

Before you do anything you'll need to install Homebrew. Copy and paste the following line into your terminal. It is incredibly important this is exact 

                
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"

Step 1.1: Update Git

MacOS already comes with a verision of Git, but you should update to the latest version. Type the following into the termianl

                
    brew install git

This will install the latest verision of Git.

Step 2: Configure Git and Github

Step 2.1: Setup Git

For Git to be able to do it's magic and work properly, we need to let it know who we are so that it can link a local Git user to GitHub. When working on a team, this allows for fellow developers to see what you have committed and who commited each line of code.

The commands below will configure Git. Be sure to edit to the correct information according.

          
    git config --global user.name "Your Name"
    git config --global user.email "yourname@example.com"

To enable a colourful output with git, type the following into your command line

        
    git config --global color.ui auto

To verify things are working properly, enter these commands and verify that the outputs matches your name and email address

        
    git config --get user.name
    git config --get user.email

Step 2.2: Create a GitHub Account and sign in

If you haven't already created an account, you can do so by following the account set-up link here. You don't have to use the same email address you used before, but it might be good idea to use the same one to keep things simple.

Step 2.3: Create an SSH Key

An SSH key is a cryptographically secure identifier. It's like a really long password used to identify your machine. GitHub uses SSH keys to allow you to upload to your repository without having to type in your username and password everytime.

First, we need to see if you have an SSH key already installed. Type the following into the terminal.

            
    ls ~/.ssh/id_rsa.pub

If a message appears in the console containing the message "No such file or directory", then you do not yyet have an SSH key, and you will need to create one. If no message has appeared in the console output, you already have a key and can proceed to step 2.4.

To create a new SSH key, run the following command inside your terminal. The -C flag followed by your email address ensure that GitHub knows who you are.

Note: The angle brackets (< >) in the code snippet below indicate that should replace the part of the command with the appropriate information. DO NOT INCLUDE THE BRACKETS THEMSELVES IN YOUR COMMAND. 

          
    ssh-keygen -C [youremail]
  • When it prompts you for a location to save the generated key just press Enter.
  • Next, it will ask you for a password; enter one if you wish, but it's not required.

Step 2.4: Link your SSH Key with GitHub

Next, you need to make GitHub aware what your SSH key is so that you can push your code without typing in a password everytime.

First, you'll meed to open up GitHub and click on your profile in the top right corner. Then, click on Settings in the drop-down menu.

Next, on the left-hand side, click SSH and GPG keys. Then, click the green button in the top right corner that says New SSH Key. Name your key something that is descriptive enough for you to remember where it came from, such as, "John-Smiths-MacBookPro". Leave this window open while you do the next steps.

Now you need to copy your public SSH key. To do this, we're going to use a command called cat to read the file to the console. (Note that the .pub file extension is important in this case.) 

          
    cat ~/.ssh/id_rsa.pub

Once this is done you should see an output appear, which starts with ssh-rsa and ends with your email address. Highlight and copy the output using CMD+C.

Now, open up that window in GitHub in your browser from earlier and paste the key you copied into the key field. Then, click Add SSH key. Congratulations you're done! Yoou've successfully added your SSH key!

Step 2.5: Testing your key

Follow the directions in this article from GitHub to verify your SSH connection. If the output doesn't correctly match up, then go through these steps again.

Git Cheat Sheet

Creating Repositories

When starting a new repository, you only need to do it once; either locally, then push to GitHub, or by cloning an exisiting repository.

        
  $ git init
        
  Turns an existing directory into a git repository
        
        
  $ git clone [ url ]
        
  Clones (downloads) a repository that already exist on GitHub.
  This includes all the files, branches, and commits.

Branches

Branches are an important part of working with Git. Any commit you make will be made on the branch you're currently working "checked out". Use $ git status to see which branch that is.

        
  $ git branch [branch-name]
        
  creates a new branch
        
        
  $ git checkout [branch-name]
        
  Switches to the specidied branch and updates the working directory

At this point you can work on your project without risk of disrupting your already working code. When you're happy with the changes you've made you can move onto the next command.

Hint: This is a great way to experiment with new code or feature before merging with your master branch

        
 $ git merge [branch]
        
  Combines the specified branch's history into the current branch.

If you've made an error and merged your branch too soon, or with faulty code you can always revert to an old version. That's the beauty of git!

        
  $ git branch -d [branch-name]
        
  Deletes the specified branch

Synchronise Changes

Synchronise your local repository with the remote repository on GitHub.

          
    $ git fetch
          
    Downloads all history from the remote tracking branches
          
          
    $ git merge
          
    Combines the remote tracking branch into the current local branch
          
          
    $ git push
          
    Uploads all local branch commits to GitHub
          
          
    $ git pull
          
    Updates your current local working branch with all new commits
    from the corresponding remote branch on GitHub.

git pull is a comination of git fetch and git merge.

Make Changes

Browse and inspect the evolution of project files

          
    $ git log
          
    Lists verision history for the current branch
          
          
    $ git add [file]
          
    Snapshots the file in preparation for versioning
          
          
    $ git commit -m "[descriptive message]"
          
    Records file snapshots permanently in version history.

Tip: Keep your commit messages concise and relevant to the changes you made
